# 助农平台 PRD

## 1. 产品概述

### 1.1 产品定位

**助农平台**是一个连接农户与消费者的农产品供需信息对接平台。平台不涉及在线交易，而是帮助农户展示农产品信息（含产地、联系方式），消费者发布采购需求（含联系方式），实现供需双方直接对接，减少中间环节，降低农产品囤货风险。

### 1.2 目标用户

| 用户角色 | 描述 | 核心需求 |
| -------- | ---- | -------- |
| **农户** | 农产品生产者 | 上传商品信息（含产地、农产地点、联系方式），浏览消费者采购需求，直接联系消费者 |
| **消费者** | 农产品采购方 | 浏览农户商品（含产地、联系方式），发布采购需求（含联系方式），等待农户联系 |
| **管理员** | 平台管理者 | 审核商品、用户管理、需求管理、评价管理、数据统计 |

### 1.3 产品愿景

打造一个透明、高效的农产品供需对接平台，让产地信息可追溯，供需双方直接沟通，助力乡村振兴，减少农产品滞销。

## 6. 业务流程

### 6.1 商品展示与对接流程

```
农户上传商品（含产地/联系方式）→ 管理员审核 → 商品上架
→ 消费者浏览商品，看到产地和联系方式 → 消费者直接联系农户 → 线下完成交易
```

### 6.2 需求发布与对接流程

```
消费者发布需求（含联系方式）→ 需求展示在需求广场
→ 农户浏览需求，看到联系方式 → 农户联系消费者 → 线下完成交易
```

### 6.3 用户登录流程

```
用户打开页面 → 检查localStorage → 已登录显示用户名 → 未登录显示登录按钮
→ 点击登录 → 输入账号密码 → 验证 → 登录成功
```

## 7. 接口设计

| 用户登录 | POST | /api/login | 用户登录 |
| 用户注册 | POST | /api/register | 用户注册 |
| 用户列表 | GET | /api/users | 获取所有用户 |
